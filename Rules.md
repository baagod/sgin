# AI 辅助开发准则 (Rules.md)

## 核心准则

### 1. 读取先行 Read-First

- 在对任何文件提出修改建议或执行操作前，**必须** 先读取该文件的最新内容。
- 严禁基于历史记忆或上下文缓存直接生成代码，以防止覆盖用户在本地进行的、未同步的修改。

### 2. 实时基准 Real-time Base

- 所有对代码的理解、分析和修改操作必须基于文件的 **真实、实时** 状态。
- 在执行命令或修改前，应通过 `read` 工具核实当前环境的物理状态，不得凭借记忆进行推断。

### 3. 尊重变更 Respect Changes

- 绝对禁止擅自恢复或回滚到代码的旧版本。如果发现当前代码与 “记忆中” 不符，应以当前物理文件内容为准。
- 意识到用户可能随时手动修改文件，AI 不得在后续操作中无意中覆盖或撤销用户的这些修改。

### 4. 最小化编辑 Minimal Editing

- 遵循 “实用主义” 原则，仅修复必要代码，严禁改动或“修复”其他未要求的部分。
- 严禁对不相关或未要求的代码块进行重构、格式化或恢复。
- 对于用户主动修改过的逻辑，除非存在明确的功能缺陷且已沟通，否则严禁自行恢复到之前的形态。
